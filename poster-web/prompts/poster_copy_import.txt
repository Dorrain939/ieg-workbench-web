【证据驱动的智能识别规则】
═══════════════════════════════════════════
- 主标题：从原文最显著标题、文件开头标题、活动名称中选择，必须使用原文措辞。
- 副标题：只在原文有明显副标题、slogan、对象说明或价值说明时输出，必须使用原文措辞。
- 原文有明确模块标题时，尽量保留原文标题；不要为了显得高级而改写标题。
- 允许把明显同义的表格类标题映射为“课程安排”或“课程内容”，例如“议程 / 排期 / 日程 / 阶段安排 / 课程表”。这是为了触发表格模块，不是语义创作。
- 允许把明显讲师类标题映射为“讲师团”，例如“讲师 / 嘉宾 / 导师 / 分享人 / 专家介绍 / 讲师阵容”。这是为了触发讲师模块，不是语义创作。
- 普通段落保持段落，不要强行拆 bullet。原文已经是列表、编号、条目时，保留为列表。
- 表格识别必须积极：原文出现多行重复结构、时间+主题、阶段+内容、日期+安排、人员+身份、课程+讲师，即使不是 Markdown 表格，也要整理成 Markdown 表格。
- 复杂表格识别必须积极：原文出现多层课程框架、阶段/模块/课程/内容多级关系、列数较多、单元格内容较长、明显跨层级重复字段时，也先整理成 Markdown 表格；后续系统会把它升级成复杂表格组件。
- 讲师识别必须积极：原文出现姓名 + 职务/身份/介绍/负责课程时，要整理成 `- 姓名 / 职务或身份 / 介绍`。缺哪项就少写哪项，不要补。
- 联系方式、报名信息、地点、名额等，如果原文已有标题就保留原标题；没有标题时，放在原顺序附近，不要抽象成新卖点。

═══════════════════════════════════════════
【硬规则】
═══════════════════════════════════════════
- 不得自由创作，不得新增原文不存在的事实。
- 不得总结、润色、换一种说法表达原文。
- 不得把普通段落改写成新的“亮点/收益/价值主张”。
- 允许的改动仅限结构化排版：标题层级、列表符号、表格列名、表格行列整理、讲师条目格式。
- 表格列名优先使用原文；原文没有列名但行结构清晰时，可以补中性列名，如“时间 / 内容 / 形式 / 讲师 / 备注”。列名只是结构标签，不得新增事实。
- 不要输出 “[待补充]”“待定”“未知”等占位，除非原文里本来就这么写。
- 不要为了凑模块而新增模块。
- 不要输出 JSON，不要输出 brief，不要输出任何解释。
