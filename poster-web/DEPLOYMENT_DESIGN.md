## 一、产品定位

**一句话**：让运营/HR/培训同事不依赖设计师，5 分钟自己拼一张合格的训练营 / 技术分享 / 文化活动海报。

**当前能力（v0.11.0）**：
- 12 种海报模块拖拽拼搭（顶部 logo、艺术字、段落、表格、讲师团、注意事项等）
- 16 套预设配色方案 + 11 个现成模板
- 实时预览（改完 1.5s 自动刷新缩略图）
- LLM 自然语言改海报（"改成深空蓝"/"副标题改成XX"）
- 输出 1440 × N 长图 PNG + PDF

**已落地海报案例**（本部门生产）：
- 腾讯游戏 2026 特效训练营 Unity 版 / UE 版
- 腾讯游戏 IEG 技术分享周（设计稿）

## 四、待改造工程清单（按优先级）

### 🔴 P0 · 必须改才能部署（约 5-7 人天）

#### 1. CDN 资源本地化
**问题**：`static/index.html` 引用 `unpkg.com/vue@3` 和 `cdn.jsdelivr.net/sortablejs`，企业内网通常封外网。
**改动**：
- 下载 `vue.global.prod.js`、`Sortable.min.js` 到 `static/vendor/`
- HTML 改为本地引用
**工作量**：2 小时

#### 2. SSO 鉴权
**问题**：当前无登录态，谁打开链接都能用，不符合内部产品要求。
**改动**：
- FastAPI 加 SSO 中间件（按公司方案选 RTX / 企微 / 腾讯统一身份）
- 未登录用户跳转 SSO，回调后写 cookie session
- 后端从 cookie 取当前用户 RTX 名作为 user_id
**工作量**：2-3 天（视 SSO 文档完善度）

#### 3. 多用户数据隔离
**问题**：当前所有用户共用 `outputs/`、`uploads/`、`localStorage` 草稿，多人会冲。
**改动**：
- `outputs/<user_id>/<job_id>/` 按用户分目录
- `uploads/<user_id>/<filename>` 按用户分目录
- 草稿改为后端持久化（用户切机器还能恢复），或保留 localStorage 但 key 加 user_id
**工作量**：1 天

#### 4. 接腾讯内部 LLM 网关
**问题**：当前调本地 `ollama`，内网服务器不会跑 ollama。
**改动**：
- 改 `api.py` 的 `/api/chat` 改为调腾讯 LLM 网关 HTTP API
- 适配 OpenAI 风格的 function_call → 网关的 tool 格式
- 鉴权头加上腾讯内部签名
**工作量**：1-2 天（看网关文档）

#### 5. 进程管理 + HTTPS 反代
**改动**：
- 写 `systemd` unit 文件（`/etc/systemd/system/poster-web.service`）
- 配 nginx 反代（80/443 → 8765，开 HTTPS）
- 申请内网域名 + SSL 证书
**工作量**：1 天

---

### 🟡 P1 · 强烈建议（约 3-5 人天）

#### 6. 渲染异步化 + 队列
**问题**：当前同步渲染，多人并发时后请求会等。
**改动**：
- 用 `concurrent.futures.ThreadPoolExecutor` 限制并发数（如 3）
- 超出排队，前端轮询状态
**工作量**：1 天

#### 7. 日志规范化
**改动**：
- 接入部门 / 公司日志平台
- 区分 access log / business log
- 关键事件（生图失败、LLM 调用错误）打 ERROR 级别
**工作量**：半天

#### 8. 资源限额
**改动**：
- 单用户每分钟最多 N 次渲染
- 单次上传文件 ≤ 10MB
- LLM 调用每用户每天 ≤ 50 次
**工作量**：半天

#### 9. 草稿持久化
**改动**：
- 当前 localStorage，换电脑就丢
- 改为后端 `drafts/<user_id>/<draft_id>.json`
- 提供"我的草稿"列表
**工作量**：1 天

## 六、部署流程

```bash
# 1. 拉代码
cd /opt && git clone <内部 git 仓库> poster-web

# 2. 装依赖
python3 -m venv venv && source venv/bin/activate
pip install -r requirements.txt

# 3. 拷贝 skill
cp -r ~/.codebuddy/skills/gaming-training-poster /opt/poster-web/skill

# 4. 配置环境变量（生产配置）
cat > /opt/poster-web/.env <<EOF
ENV=production
DATA_DIR=/data/poster-web
LLM_GATEWAY_URL=https://llm-gateway.intra.tencent.com/api/v1
LLM_API_KEY=...
SSO_PROVIDER=rtx
EOF

# 5. 启动 systemd
sudo cp poster-web.service /etc/systemd/system/
sudo systemctl enable poster-web && sudo systemctl start poster-web

# 6. 配置 nginx
sudo cp nginx.conf /etc/nginx/sites-available/poster-web
sudo ln -s ... && sudo nginx -t && sudo systemctl reload nginx
```

## 七、安全与合规

### 7.1 数据安全
- 用户上传素材仅本人可见（按 user_id 鉴权访问）
- 渲染产物 7 天后自动清理
- 不收集生成内容，仅记录"谁在何时调用了何 API"

### 7.2 合规要点
- ✅ 字体：腾讯体仅限内部使用，部署在内网完全合规
- ✅ 用户数据：仅 RTX 名 + cookie session，不涉及敏感个人信息
- ✅ 海报内容：用户自传素材，平台不审核内容（与 PPT 工具类似）
- ⚠️ LLM 输出：通过腾讯内 LLM 网关，自带合规过滤

### 7.3 访问控制
- 仅 IEG 域账号可登录（SSO 限定 OU）
- 管理员可在后台禁用某用户

## 附录 A：术语表

- **brief.json**：海报描述文件，定义画布配置 + 各 section 的内容/样式
- **section**：海报中的一个内容模块，如「主标题艺术字」「讲师团方阵」
- **skill 引擎**：底层 PIL 渲染逻辑，输入 brief.json 输出 PNG/PDF
- **session**：用户的一次浏览器会话，隔离草稿和上传素材
- **palette_strategy**：配色方案标识，如 `named:cyber_neon` 表示霓虹赛博紫
- **chroma_key**：图片抠图，把背景色变透明
- **ollama / qwen2.5:14b**：开源本地 LLM 框架 / 阿里通义千问 14B 参数模型
